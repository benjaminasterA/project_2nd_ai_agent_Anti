# [행동 요령] AI 인사이트 리포터 개발 가이드

## 1. 개발 착수 전 (Preparation)
- [ ] **수집 대상 선정**: 수집할 뉴스 사이트의 RSS 피드 또는 URL 목록을 정의합니다.
- [ ] **저작권 가이드 확인**: 데이터 수집 시 법적 저작권 문제를 방지하기 위해 Robots.txt 및 이용 약관을 준수합니다.

## 2. 단계별 행동 요령 (Step-by-Step Action)

### Phase 1: 스크래핑 파이프라인
- 정기적으로 정보를 가져오는 정기 스케줄러(APScheduler 등)를 구축합니다.
- 뉴스 포맷이 달라도 동일한 스키마(제목, 본문, 일자)로 저장할 수 있는 파서를 만듭니다.
- **Backend Port**: 8001번을 기본으로 사용합니다.

### Phase 2: 클러스터링 및 중복 기사 제거
- TF-IDF 또는 Sentence Embedding을 사용하여 의미적으로 유사한 기사를 묶습니다.
- 가장 정보량이 많은 기사를 대표 기사로 선정하고 나머지는 요약 시 참조용으로 활용합니다.

### Phase 3: 멀티 LLM 요약 노하우
- 비용 절감을 위해 초안 요약은 경량 모델(GPT-4o-mini)을 사용하고, 최종 인사이트 검증만 상위 모델을 검토합니다.
- 요약문의 일관성을 위해 시스템 프롬프트(System Prompt)를 구조적으로 관리합니다.

## 3. 유지보수 및 기록 (Maintenance)
- `HISTORY.md`를 업데이트하여 필터링 알고리즘이 개선될 때마다 기록합니다.
- 수집 실패 시 알람(Slack/Email)을 받을 수 있는 모니터링 로직을 추가할 것을 권장합니다.
